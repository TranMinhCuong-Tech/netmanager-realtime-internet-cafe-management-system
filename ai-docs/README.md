# AI Docs

This folder is the persistent working memory for the project.

## Read First

1. [PROJECT_STATE.md](./PROJECT_STATE.md)
2. [TASKS.md](./TASKS.md)
3. [ARCHITECTURE.md](./ARCHITECTURE.md) if you need system context
4. [API.md](./API.md) if you are working on packets or contracts
5. [LEADER_FLOW.md](./LEADER_FLOW.md) if you are coordinating sequence and ownership

## Update When You Finish Work

- [PROJECT_STATE.md](./PROJECT_STATE.md)
- [TASKS.md](./TASKS.md)
- [BUGS.md](./BUGS.md) if new issues appear
- [ARCHITECTURE.md](./ARCHITECTURE.md) if structure changes
- [API.md](./API.md) if packet or schema contracts change
- [DECISIONS.md](./DECISIONS.md) if an important technical choice is made
- [LEADER_FLOW.md](./LEADER_FLOW.md) if task order or ownership changes

## Purpose

- keep the project state easy to resume across sessions
- avoid rereading the whole codebase every time
- preserve decisions, known issues, and next steps in one place

## Suggested Daily Flow

1. Open `PROJECT_STATE.md`.
2. Open `TASKS.md`.
3. Continue the latest unfinished task.
4. Update the memory files after the session ends.

## Notes

- Keep entries short and current.
- Remove outdated information instead of duplicating it.
- Treat these files as the source of truth for project continuity.
